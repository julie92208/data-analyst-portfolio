Fitness App Database System

MySQL Project by Ya-Jwu Jang | Conestoga College

Project Overview

A complete MySQL database system designed for a fitness tracking app. The system allows users to monitor workouts, track progress, and manage subscriptions. It demonstrates data normalization, DML/DDL scripting, stored procedures, and database backup techniques.

My Role

DML & Stored Procedure Developer

  • Created full DML scripts to populate all eight tables with realistic data.
  • Designed a stored procedure (listlazyusers) to identify inactive users with active subscriptions.
  • Performed complete database backup using mysqldump.

Project Goal

Business Objective: Detect users who have not been active since a specified date but still hold active memberships. This logic supports user re-engagement and retention campaigns. 

CALL listlazyusers('2024-11-01');

Result: 6 users identified as inactive but subscribed.

Procedure Example

Goal: List customers who have not been active since a given date and are still subscribed.

Input: 2024-11-01 ‚Üí Result: 6 users


SELECT 
    user.UserID,
    user.Fname,
    user.Lname,
    user.Email,
    MAX(progress.Date) AS LastTime_Workout,
    MAX(Transaction.Startdate) AS Subscription_startdate,
    discount.duration,
    DATE_ADD(MAX(Transaction.Startdate), INTERVAL discount.duration DAY) AS subscription_enddate
FROM user
JOIN progress ON progress.userID = user.userID
JOIN Transaction ON Transaction.User_UserID = user.UserID
JOIN discount ON discount.DiscountID = Transaction.Discount_DiscountID
WHERE progress.date < '2024-09-01'
GROUP BY UserID
HAVING subscription_enddate > CURRENT_DATE()
ORDER BY UserID;
